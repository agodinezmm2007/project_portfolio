C.3 Queries for Study Selection and Categorization (PRISMA Flow)

The following queries were instrumental in the study selection process detailed in the PRISMA flow diagram. These queries implement the mutually exclusive categorization of articles.

C.3.1 Identifying Overlapping Categories in Initial Analysis

An initial step to refine categorization logic involved identifying articles that could be ambiguously classified. The following query identified articles that possessed characteristics of both non-empirical and empirical studies (and lacked ML methods), highlighting the need for a prioritized classification scheme:

MATCH (a:Article)
// Condition for having at least one Non-Empirical Study Type
WHERE EXISTS {
  MATCH (a)-[:STUDY_TYPE]->(st_ne:StudyType)
  WHERE toLower(st_ne.name) IN [
    'review','systematic review','meta-analysis','expert opinion',
    'scoping review','dissertation/thesis','short communication',
    'methodological paper','theoretical study','report'
  ]
}
// Condition for having at least one Empirical Study Type
AND EXISTS {
  MATCH (a)-[:STUDY_TYPE]->(st_e:StudyType)
  WHERE NOT toLower(st_e.name) IN [
    'review','systematic review','meta-analysis','expert opinion',
    'scoping review','dissertation/thesis','short communication',
    'methodological paper','theoretical study','report'
  ]
}
// Condition for being part of EmpiricalWithoutML (Q2)
AND NOT EXISTS((a)-[:USES_ML_METHOD]->(:MLMethod))
RETURN COUNT(DISTINCT a.doi) AS OverlappingArticles_Q1_and_Q2

C.3.2 Comprehensive Mutually Exclusive Categorization Query

This query assigns each of the 439 articles to a unique category, forming the basis for the PRISMA flow diagram exclusions. Non-empirical classifications are prioritized.

MATCH (a:Article)

// Collect all study type names for each article
OPTIONAL MATCH (a)-[:STUDY_TYPE]->(st_all:StudyType)
WITH a, COLLECT(DISTINCT toLower(st_all.name)) AS studyTypeNames

// Determine if article has any non-empirical or empirical study types based on collected names
WITH a, studyTypeNames,
     [sName IN studyTypeNames WHERE sName IN [
       'review','systematic review','meta-analysis','expert opinion',
       'scoping review','dissertation/thesis','short communication',
       'methodological paper','theoretical study','report'
     ]] AS nonEmpiricalSTList,
     [sName IN studyTypeNames WHERE sName <> '' AND sName IS NOT NULL AND NOT sName IN [
       'review','systematic review','meta-analysis','expert opinion',
       'scoping review','dissertation/thesis','short communication',
       'methodological paper','theoretical study','report'
     ]] AS empiricalSTList

// Gather other relevant existence checks
WITH a, studyTypeNames, nonEmpiricalSTList, empiricalSTList,
     EXISTS((a)-[:USES_ML_METHOD]->(:MLMethod)) AS hasML,
     EXISTS((a)-[:ASSOCIATED_WITH_HEART_DISEASE]->(:HeartDisease)) AS hasHD,
     EXISTS { 
       MATCH (a)-[:RELATED_TO_POLLUTANT]->(p:PollutantTerm) 
       WHERE toLower(p.name) CONTAINS 'ozone' 
     } AS hasOzone

// Assign a single category to each article
WITH a,
     CASE
       WHEN size(studyTypeNames) = 0 OR (size(studyTypeNames)=1 AND studyTypeNames[0] IS NULL) THEN 'A_MissingOrNullStudyType'
       WHEN size(nonEmpiricalSTList) > 0 THEN 'B_NonEmpirical' // Priority: if any ST is non-empirical
       WHEN size(empiricalSTList) > 0 THEN // Only if no non-empirical STs, then check for empirical
         CASE
           WHEN NOT hasML THEN 'C_EmpiricalWithoutML'
           ELSE // Empirical AND HasML
             CASE
               WHEN hasHD AND hasOzone THEN 'D_EmpiricalML_Eligible_OzoneHD'
               ELSE 'E_EmpiricalML_NotEligible_OzoneHD'
             END
         END
       ELSE 'F_StudyType_Other_Or_Invalid' // Has study types, but not categorized (e.g., empty names not caught by null check)
     END AS finalCategory
RETURN finalCategory, COUNT(DISTINCT a.doi) AS count
ORDER BY finalCategory

C.4 Characterization of Included Studies (n=34)

Following the comprehensive categorization detailed in Section D.3, 36 articles were initially identified as potentially eligible (Category D_EmpiricalML_Eligible_OzoneHD). Upon closer inspection of these 36 articles, a further two were excluded as they were determined to be commentary or reply-to-commentary articles, rather than primary research. This resulted in a final set of 34 unique studies included in this systematic review.

The Cypher query used to extract detailed characteristics for these 34 studies (after applying title-based filters for comments/replies) is presented below. The full tabular output of this query, listing each of the 34 studies and their associated attributes, is available in the project’s data repository at data/export.csv.

// Collect all study type names per article
OPTIONAL MATCH (a)-[:STUDY_TYPE]->(st:StudyType)
WITH a, COLLECT(DISTINCT toLower(st.name)) AS studyTypeNames

// Only include articles with at least one empirical StudyType (and no non-empirical types)
WHERE SIZE(studyTypeNames) > 0
  AND NONE(stName IN studyTypeNames WHERE stName IN [
    'review', 'systematic review', 'meta-analysis', 'expert opinion',
    'scoping review', 'dissertation/thesis', 'short communication',
    'methodological paper', 'theoretical study', 'report'
  ])

// Check for ML methods, heart disease, and ozone pollutant explicitly
AND EXISTS((a)-[:USES_ML_METHOD]->(:MLMethod))
AND EXISTS((a)-[:ASSOCIATED_WITH_HEART_DISEASE]->(:HeartDisease))
AND EXISTS {
  MATCH (a)-[:RELATED_TO_POLLUTANT]->(p:PollutantTerm)
  WHERE toLower(p.name) CONTAINS 'ozone'
}

// Exclude comment/reply-type articles explicitly
AND NOT toLower(a.title) CONTAINS 'comment'
AND NOT toLower(a.title) CONTAINS 'reply'

// Retrieve related information
OPTIONAL MATCH (a)-[:RELATED_TO_POLLUTANT]->(pt:PollutantTerm)
OPTIONAL MATCH (a)-[:USES_ML_METHOD]->(ml:MLMethod)
OPTIONAL MATCH (a)-[:ASSOCIATED_WITH_HEART_DISEASE]->(hd:HeartDisease)
OPTIONAL MATCH (a)-[:STUDY_TYPE]->(stypes:StudyType)
OPTIONAL MATCH (a)-[:USES_ANALYTICAL_TOOL]->(at:AnalyticalTool)

WITH DISTINCT a.doi AS DOI,
              a.title AS Title,
     COLLECT(DISTINCT stypes.name) AS StudyTypes,
     COLLECT(DISTINCT pt.name) AS Pollutants,
     COLLECT(DISTINCT ml.name) AS MLMethods,
     COLLECT(DISTINCT hd.name) AS HeartDiseases,
     COLLECT(DISTINCT at.name) AS AnalyticalTools

RETURN DOI,
       Title,
       StudyTypes,
       Pollutants,
       MLMethods,
       HeartDiseases,
       AnalyticalTools
ORDER BY DOI

C.4.1 Distribution of Study Types

The 34 included empirical studies exhibited a range of specific study designs. The frequency of each study type is presented below. (Note: Articles may be associated with multiple study type labels if applicable from the source data, though the KG processing typically unifies these to canonical forms; the table reflects the primary unified types found across the 34 studies).

MATCH (a:Article)

// Collect all study type names per article
OPTIONAL MATCH (a)-[:STUDY_TYPE]->(st:StudyType)
WITH a, COLLECT(DISTINCT toLower(st.name)) AS studyTypeNames

// Only include articles with at least one empirical StudyType (and no non-empirical types)
WHERE SIZE(studyTypeNames) > 0
  AND NONE(stName IN studyTypeNames WHERE stName IN [
    'review', 'systematic review', 'meta-analysis', 'expert opinion',
    'scoping review', 'dissertation/thesis', 'short communication',
    'methodological paper', 'theoretical study', 'report'
  ])

// Check for ML methods, heart disease, and ozone pollutant explicitly
AND EXISTS((a)-[:USES_ML_METHOD]->(:MLMethod))
AND EXISTS((a)-[:ASSOCIATED_WITH_HEART_DISEASE]->(:HeartDisease))
AND EXISTS {
  MATCH (a)-[:RELATED_TO_POLLUTANT]->(p:PollutantTerm)

}

// Exclude comment/reply-type articles explicitly
AND NOT toLower(a.title) CONTAINS 'comment'

// Retrieve and unify study types

C.4.6 Cypher Queries Cheat Sheet

To get a list of metrics from every DOI:

MATCH (a:Article)-[r:REPORTS_METRIC]->(m:Metric)
RETURN  a.doi      AS DOI,
        r.MetricID AS metricid,
        m.name     AS metric,
        r.value    AS value,
        r.unit     AS unit
ORDER BY DOI, metricid;

to get metrics associated with any given study type (for example cohort study)

/* Cohort-study articles and any metrics they report */
MATCH (a:Article)-->(:StudyType {name: "cohort study"})
OPTIONAL MATCH (a)-[r:REPORTS_METRIC]->(m:Metric)

RETURN
  "cohort study"                               AS studyType,
  a.doi                                        AS DOI,
  coalesce(r.MetricID , "NO METRICS")          AS metricid,
  coalesce(m.name     , "NO METRICS")          AS metric,
  coalesce(r.value    , "NO METRICS")          AS value,
  coalesce(r.unit     , "NO METRICS")          AS unit
ORDER BY DOI, metricid;

or if you are interested in life cycle assessment:

MATCH (a:Article)-->(:StudyType {name: "life cycle assessment"})
OPTIONAL MATCH (a)-[r:REPORTS_METRIC]->(m:Metric)

RETURN
  "life cycle assessment"                               AS studyType,
  a.doi                                        AS DOI,
  coalesce(r.MetricID , "NO METRICS")          AS metricid,
  coalesce(m.name     , "NO METRICS")          AS metric,
  coalesce(r.value    , "NO METRICS")          AS value,
  coalesce(r.unit     , "NO METRICS")          AS unit
ORDER BY DOI, metricid;

to get a list and count of all nodes:

MATCH (n)
UNWIND labels(n) AS nodeLabel
RETURN nodeLabel, COUNT(*) AS nodeCount
ORDER BY nodeCount DESC;

to get a list and count of all relationships:

MATCH ()-[r]->()
RETURN TYPE(r) AS relType, COUNT(*) AS relCount
ORDER BY relCount DESC;
